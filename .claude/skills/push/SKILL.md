오늘 작업 내용을 CLAUDE.md에 정리하고, GitHub에 자동 커밋·푸시합니다.

## 실행 순서

### Step 1 — 변경 내용 파악

`git status`, `git diff`, `git log --oneline -10`을 실행해서 오늘 변경된 파일과 내용을 파악하세요.

### Step 2 — CLAUDE.md 개발 일지 업데이트

CLAUDE.md의 `## 개발 일지` 섹션을 찾아서 **오늘 날짜(YYYY-MM-DD)** 항목을 추가하거나 업데이트하세요.

- 이미 오늘 날짜 항목이 있으면 내용을 보완하세요.
- 항목이 없으면 가장 최근 날짜 항목 위에 새로 추가하세요.
- 작성 형식:

```markdown
### YYYY-MM-DD

**완료 작업**:
- `파일경로` — 핵심 변경 내용 한 줄 요약 (함수명, 기능 포함)

**미확정 사항** (있을 경우만):
- 아직 결정되지 않은 사항
```

- 내용은 **한국어**로, 기술 용어(함수명·파일명)는 원문 유지
- `## 개발 일지` 섹션이 없으면 `## 도메인 용어` 섹션 바로 위에 새로 만드세요.

### Step 3 — Commit & Push

1. 커밋 메시지를 한국어로 작성하세요.
   - 형식: `타입: 간결한 설명` (예: `feat: GR/IR 매칭 엔진 초기 구현`)
   - 타입: feat(새 기능) / fix(버그수정) / refactor(리팩터링) / test(테스트) / docs(문서) / chore(기타)
   - 인자 `$ARGUMENTS`가 있으면 그것을 커밋 메시지로 사용하세요.
2. 변경 파일을 스테이징하세요 (`.env`, `*.key`, `*secret*` 파일은 제외).
3. 커밋을 생성하세요.
4. `git push origin HEAD`로 현재 브랜치에 푸시하세요.
5. 완료 후 커밋 해시와 푸시된 브랜치를 출력하세요.

## 주의사항
- `.env`, `*.key`, `*secret*` 파일은 절대 커밋하지 마세요.
- 변경 사항이 없으면 "커밋할 변경 사항이 없습니다"라고 알려주세요.
- CLAUDE.md 업데이트도 커밋에 포함하세요.
