# 재고매입결산 자동화 Plan

## Context

회계팀 재고매입결산 담당자가 매 결산 시 SAP에서 GR/IR 데이터를 추출한 뒤,
PO ↔ GR ↔ IR 3-way 매칭을 수동으로 검증하고 잔액을 미착품/미확정채무로
계정 분류하는 반복 업무를 자동화한다.

현재 `sap_excel_cleaner.py`가 SAP Excel 정제 기능을 제공하고 있으므로,
이를 재사용하여 GR/IR 매칭 엔진을 새 모듈로 추가한다.

---

## 업무 흐름 요약

```
[구매팀]                          [회계팀]
  ① PO 생성
  ② 입고처리 (GR)
  ③ 송장처리 (IR)
  └─────────────────▶ ④ GR/IR 매칭 검증
                         ⑤ 잔액 분류 (미착품 / 미확정채무 / 예외)
                         ⑥ 결과 리포트 출력 → 전표 작성 (수동)
```

---

## 자동화 vs 수동 검수 경계

| 단계 | 내용 | 구분 |
|------|------|:----:|
| SAP Excel 정제 | 헤더 제거, 빈행 삭제 | ✅ 자동 |
| PO 기준 GR·IR 조인 | pandas merge | ✅ 자동 |
| 완전 매칭 분류 | 잔액 = 0 | ✅ 자동 |
| 미착품 분류 | GR 있음 · IR 없음 | ✅ 자동 |
| 미확정채무 분류 | IR 있음 · GR 없음 | ✅ 자동 |
| Aging 계산 | 오늘 - PO 일자 | ✅ 자동 |
| 예외 건 검토 | 금액 불일치, 부분입고 | ⚠️ 수동 |
| 오래된 미결 건 검토 | Aging 임계값 초과 | ⚠️ 수동 |
| 외화 환율 차이 건 | 환산 시점 판단 | ⚠️ 수동 |
| 최종 전표 승인 | 내부통제 요건 | ⚠️ 수동 |

### gr_ir_matcher.py 구조

```python
# 1. 데이터 로드
load_gr_data(path)   # GR 추출 Excel 로드 후 정제
load_ir_data(path)   # IR 추출 Excel 로드 후 정제

# 2. 매칭 엔진
match_gr_ir(gr_df, ir_df)
  → PO 번호 기준 left outer join
  → 잔액 = GR 금액 합계 - IR 금액 합계 (PO별 groupby)
  → 분류 컬럼 추가:
      잔액 == 0          → "완전매칭"
      GR 있음, IR 없음   → "미착품"
      IR 있음, GR 없음   → "미확정채무"
      금액 불일치        → "예외_금액차이"
      수량 불일치        → "예외_수량차이"

# 3. Aging 계산
add_aging(df)
  → 오늘 날짜 - PO 일자 = 경과일수
  → 구간: ~30일 / 31~60일 / 61~90일 / 90일 초과

# 4. 리포트 생성
export_report(df, output_path)
  → 시트 분리:
      [완전매칭] [미착품] [미확정채무] [예외건] [전체]
  → openpyxl 서식 적용 (sap_excel_cleaner._apply_styles 참고)
  → 예외건 시트에 "검토의견" 빈 컬럼 추가
```

---

### 매칭 로직 상세

```
입력 컬럼 (최소 요구):
  GR 파일: PO번호, 입고일, 자재코드, 입고수량, 입고금액, 통화
  IR 파일: PO번호, 송장일, 자재코드, 송장수량, 송장금액, 통화

분류 우선순위:
  1. PO번호별 GR합계 vs IR합계 계산
  2. 잔액 = GR합계 - IR합계
     - abs(잔액) <= 허용오차(기본 1원) → 완전매칭
     - GR합계 > 0, IR합계 == 0        → 미착품
     - IR합계 > 0, GR합계 == 0        → 미확정채무
     - 그 외                           → 예외
```

## 결과물 예시 (Excel 리포트)

| 시트 | 내용 |
|------|------|
| 완전매칭 | 잔액 0, 처리 완료 건 |
| 미착품 | GR완료·IR미도착 → 자산 계정 처리 대상 |
| 미확정채무 | IR완료·GR미완 → 부채 계정 처리 대상 |
| 예외건 | 금액/수량 불일치, 검토의견 입력란 포함 |
| 전체 | 위 4개 합본 |

---

## 검증 방법

1. SAP MB5S 트랜잭션 조회 결과와 리포트 합계 금액 대사
2. 완전매칭 건수 + 미착품 건수 + 미확정채무 건수 + 예외 건수 = 전체 PO 건수 확인
3. 미착품 합계 = GR/IR 계정 차변 잔액, 미확정채무 합계 = 대변 잔액 일치 여부 확인

## 시작 전 확인 필요 사항

- SAP에서 추출하는 GR/IR Excel의 실제 컬럼명 확인 필요
  (PO번호, 금액, 수량 컬럼이 SAP 언어 설정에 따라 다를 수 있음)
- 부분입고(부분 GR) 처리 방식: PO 라인 단위 vs PO 번호 단위 집계 선택
- 외화 PO 환율 처리: 원화 환산 기준 시점 (입고일 vs 송장일 vs 결산일)
